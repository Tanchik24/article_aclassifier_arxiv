# Отчет по экспериментам


## Цель

Построить multilabel-классификатор arXiv-статей по `title + abstract` и предсказывать набор тематических тегов (148 категорий)

## Пайплайн экспериментов

1. Подготовка данных
2. Обучение baseline-модели на DistilBERT
3. Обучение SciBERT (`allenai/scibert_scivocab_cased`)
4. Улучшенная SciBERT-модель (`ultra_scibert`) с кастомной архитектурой и ассиметричным лоссом
5. Дополнительный эксперимент с иерархической классификацией

## Данные 

В качестве данных использовался датасет статей arXiv с Kaggle, содержащий идентификатор статьи, заголовок, аннотацию и список категорий. На этапе предобработки из разметки были оставлены только допустимые arXiv-категории, а часть тегов была приведена к каноническому виду для устранения дубликатов и пересечений между близкими классами. Далее из-за сильного дисбаланса классов датасет был дополнен новыми статьями через arXiv API для тех категорий, где число примеров было меньше 300. После объединения данных были удалены дубликаты и отфильтрованы слишком редкие классы, для которых даже после дополнения осталось меньше 300 объектов. В результате был получен очищенный и более сбалансированный датасет с 148 тегами, использованный для обучения модели.

## Эксперименты и настройки

### 1) Baseline: DistilBERT

- Базовая модель: `distilbert/distilbert-base-cased`
- Параметры:
  - `num_train_epochs=5`
  - `learning_rate=2e-5`
  - `lr_scheduler_type="cosine"`
  - `warmup_steps=5`
  - `weight_decay=0.01`
  - `per_device_train_batch_size=16`


### 2) SciBERT

- Базовая модель: `allenai/scibert_scivocab_cased`
- Параметры:
  - `num_train_epochs=3`
  - `learning_rate=2e-5`
  - `lr_scheduler_type="cosine"`
  - `warmup_steps=5`
  - `weight_decay=0.01`
  - `per_device_train_batch_size=16`
  - `metric_for_best_model="macro_f1"`


### 3) Ultra SciBERT (основная финальная версия)

- Базовая модель: `allenai/scibert_scivocab_cased`
- Кастомная голова + `AsymmetricLoss(gamma_neg=4.0, gamma_pos=1.0, clip=0.05)`
- Параметры:
  - `num_train_epochs=5`
  - `learning_rate=2e-5`
  - `per_device_train_batch_size=8`
  - `warmup_steps=200`
  - `weight_decay=0.01`
  - `gradient_accumulation_steps=2`
  - `metric_for_best_model="macro_f1"`


### 4) Hierarchical SciBERT

- Базовая модель: `allenai/scibert_scivocab_cased`
- Цель: раздельная оценка main/fine уровней таксономии
- Параметры:
  - `num_train_epochs=10`
  - `learning_rate=3e-5`
  - `per_device_train_batch_size=8`
  - `lr_scheduler_type="cosine"`
  - `warmup_steps=500`
  - `weight_decay=0.01`
  - `per_device_train_batch_size=8`
  - `gradient_accumulation_steps=2`

## Результаты 

### Baseline: DistilBERT

| Метрика | Значение |
|---|---|
| `macro_f1` | 0.325358 |
| `micro_f1` | 0.586162 |
| `macro_precision` | 0.637834 |
| `macro_recall` | 0.252285 |

Существенный разрыв между micro и macro метриками указывает на то, что модель значительно лучше распознаёт частые классы, чем редкие. При этом высокое значение precision относительно recall показывает, что модель склонна к осторожным предсказаниям и часто пропускает релевантные метки, особенно в хвосте распределения классов 


### SciBERT

| Метрика | Значение |
|---|---|
| `macro_f1` | 0.058209 |
| `micro_f1` | 0.482607 |
| `macro_precision` | 0.141898 |
| `macro_recall` | 0.047524 |

Большой разрыв между micro F1 и macro F1 подтверждает, что модель в основном лучше работает на частых классах, а на редких метках почти не обобщает. Также низкие значения macro precision и macro recall указывают на слабое качество предсказаний по хвосту распределения. Модель изначально показывала качество весьма низкое, поэтому решила остановить очение черезе 2 эпохи

### Ultra SciBERT (финальная рабочая версия)

| Метрика | Значение |
|---|---:|
| `global_threshold` | 0.6 |
| `macro_f1` | 0.5608 |
| `micro_f1` | 0.6586 |
| `macro_precision` | 0.5644 |
| `macro_recall` | 0.5696 |
| `train_loss` | 0.00919 |

Была использована более сложная конфигурация на основе SciBERT с асимметричной функцией потерь и взвешенным семплированием для компенсации дисбаланса классов. Рост значений macro F1 и macro recall показывает, что модель стала значительно лучше работать не только на частых, но и на редких классах. При этом более высокое значение recall по сравнению с precision указывает на то, что модель стала предсказывать метки смелее и лучше покрывать истинные классы. Дополнительное улучшение результатов было получено после подбора глобального порога классификации

### Hierarchical SciBERT

| Метрика | Значение |
|---|---|
| `fine_macro_f1` | 0.315077 |
| `fine_micro_f1` | 0.389045 |
| `main_macro_f1` | 0.629782 |
| `main_micro_f1` | 0.776514 |


В данном эксперименте была использована иерархическая модель на основе SciBERT, в которой одновременно предсказывались метки верхнего уровня и более детальные fine метки. Качество для fine меток оказалось заметно ниже, чем для высокоуровневых категорий, что указывает на более высокую сложность детальной классификации. При этом значения fine recall существенно выше fine precision, то есть модель стала лучше находить истинные, в том числе редкие, метки, но ценой большего числа ложноположительных предсказаний. Такой подход оказался достаточно сложным в настройке, поскольку требует подбора большего числа гиперпараметров и медленнее сходится при обучении. Основная идея заключалась в том, чтобы улучшить предсказание 148 fine-классов за счёт предварительного выделения более общего, абстрактного класса
